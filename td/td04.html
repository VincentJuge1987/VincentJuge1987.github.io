Premiers pas en Javascript
Comme pour les TP précédents, on vous demande de créer un répertoire tp4 qui contiendra les exercices de cette feuille de TP et de faire un compte rendu de TP en HTML.
Ce TP sera noté, mais vous pouvez bien sûr demander de l'aide à vos enseignants : le but de la note est de vous pousser à travailler sérieusement et de vous donner une idée de votre niveau de compréhension du cours.
Vous avez le droit de lire l'intégralité du sujet du TP pour comprendre où on souhaite que vous alliez.
Attention : Chaque fois que l'on vous demande de réécrire une fonction ou de remplacer votre code par un nouveau code, il faudra mettre l'ancien code en commentaire et surtout pas le supprimer. Sinon, votre chargé de TP ne sera plus en mesure de vérifier que l'ancien code répondait à la question précédente.

Exercice 1 : Hello World

Cet exercice montre comment utiliser les outils de developpement présents sur les navigateurs, comme la console.

  1. Recopiez la page Web exo1.html dans votre répertoire tp4.
    Que se passe-t-il lorsque l'on visualise la page ?
    Comment fait-on pour ouvrir la console avec Firefox ?
    Et avec Chrome ?
  2. Mettez l'ancien code en commentaire.
    Ecrivez un code permettant d'afficher, toujours dans la console, les chiffres de 0 à 2022 inclus (il n'est pas nécessaire de faire une fonction pour l'instant).
    Note : par pitié, pas de copier/coller !
  3. Mettez l'ancien code en commentiare.
    Réécrivez un nouveau code code pour afficher les chiffres de 0 à 2022 sauf les multiples de 3 (c'est-à-dire sauf ceux qui sont divisibles par 3).
  4. Commentez l'ancien code et écrire une fonction (que vous nommerez de façon claire et intelligente) prenant un entier en paramètre et affichant sur la console les nombres de 0 à cet entier (non compris). Puis écrivez un code de test de la fonction avec les valeurs 3 et 12.

Exercice 2 : Damier

On cherche à faire un damier avec des cases blanches et noires et la diagonale affichée en rouge.
Attention : Chaque fois que l'on souhaitera donner un style spécifique à tel ou tel élément HTML, on lui donnera une classe on un ID et on créera le sélecteur CSS correspondant : écrire des informations de style en dur dans une balise HTML, c'est une méthode de sauvages !

  1. Recopiez la page Web exo2.html dans votre répertoire tp4.
    Expliquez ce que fait la première ligne du script : 
    "use strict";
  2. Expliquez ce que fait l'appel document.write().
  3. Ajoutez une fonction drawCheckerLine qui affiche une ligne de 5 cases grâce à la méthode drawCheckerCell.
    Note : Le plus simple est de mettre les 5 boites collées à gauche à l'intérieur d'une grosse boite qui correspondra à la ligne.
  4. Ajoutez une fonction drawCheckerBoard pour afficher un carré de 5 cases par 5 cases.
  5. Commentez votre code de la fonction drawCheckerBoard, puis réécrivez une nouvelle version de cette fonction pour que, dans la page web, on puisse afficher un carré de 5 par 5 puis un carré de 3 par 3. Vérifiez que l'affichage se fait correctement.
    Note: attention au float en CSS !
  6. Complétez le style de la balise style du document pour définir une case noire et une case blanche et utilisez ce style pour les cases du damier.
    Note : une balise HTML peut avoir plusieurs classes CSS !
    Astuce : la somme des coordonnées des cases modulo 2 (le reste de la division par 2) vous indique s'il s'agit d'une case blanche ou d'une case noire.
  7. Ajoutez un style de case rouge puis commentez votre code de la fonction drawCheckerBoard et réécrivez-la pour ajouter un dernier paramètre à drawCheckerBoard pouvant être vrai ou faux. Si le paramètre est vrai, alors il faudra afficher la diagonale en rouge.
    Au cours de ce processus, vous pourrez être amené à commenter les code des fonctions drawCheckerCell et/ou drawCheckerLine pour leur ajouter des paramètres additionnels que vous aurez jugés pertinents.
  8. Testez en affichant la diagonale du premier carré (pas celle du second).

Exercice 3 : On fera en fonction…

On peut améliorer la "beauté" du code, et le rendre un peu plus professionnel, en évitant de passer en paramètre un booléen pour dire s'il faut dessiner ou non la diagonale.
  1. Copiez-collez le document exo2.html et nommez exo3.html votre copie. Ci-dessous, vous effectuerez vos modifications sur ce nouveau document exo3.html.
  2. Assurez vous que la fonction drawCheckerCell prenne un argument, qui sera la couleur de la case correspondante.
    Par exemple, si j'appelle drawCheckerCell(5, "red"), on dessinera une case de classe "red", qui aura vocation à être dessinée en rouge comme indiqué dans les informations de style du document.
  3. Redéfinissez la fonction drawCheckerLine pour que, outre la longueur de la ligne qu'elle doit dessiner, le deuxième argument que prend cette fonction soit non plus le numéro de la ligne correspondante, mais une fonction d'affichage qui associera à chaque entier j la couleur de la case n°j sur cette ligne.
    Par exemple, si j'appelle drawCheckerLine(5, fun), on dessinera 5 cases de couleurs fun(0), fun(1), fun(2), fun(3) et fun(4).
  4. Redéfinissez la fonction drawCheckerBoard pour que, outre la dimension de la grille qu'elle doit dessiner, le deuxième argument que prend cette fonction soit non plus un booléen qui dira si on doit peindre (ou non) en rouge la diagonale de la grille, mais une fonction d'affichage qui associera à paire (i,j) la couleur de la case en ligne i et colonne j.
    Par exemple, si j'appelle drawCheckerBoard(3, fun), on dessinera 3×3 cases, et la case en ligne 1 et colonne 2 sera de couleur fun(1,2).
  5. Définissez une fonction diagonalColor dont le seul argument est un booléen, et telle que l'appel à drawCheckerBoard(size, diagonalColor(true)) dessinera une grille dont la diagonale sera rouge, tandis que l'appel à drawCheckerBoard(size, diagonalColor(false)) dessinera une grille dont la diagonale n'aura pas de couleur particulière (par rapport aux autres cases de la grille).
